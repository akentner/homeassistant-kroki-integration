  <action>
    Add service registration to `async_setup`. The handler resolves the targeted entity via the
    HA entity component and dispatches to `async_force_render()`.

    **Service schema:** Use `vol.Schema({vol.Optional(ATTR_ENTITY_ID): cv.entity_ids})` — this
    accepts a list of entity IDs. ATTR_ENTITY_ID is from `homeassistant.const`.

    **Handler logic (per D-06, D-07, D-08, D-09):**

    ```python
    async def _async_handle_force_render(call: ServiceCall) -> None:
        """Handle the kroki.force_render service call."""
        entity_ids: list[str] = call.data.get(ATTR_ENTITY_ID, [])
        # Resolve the image entity component to look up live entities
        component = hass.data.get("entity_components", {}).get(Platform.IMAGE)
        for entity_id in entity_ids:
            if component is None:
                _LOGGER.warning(
                    "force_render: image entity component not available, skipping %s", entity_id
                )
                continue
            entity = component.get_entity(entity_id)
            if entity is None or not isinstance(entity, KrokiImageEntity):
                _LOGGER.warning(
                    "force_render: entity %s not found or not a KrokiImageEntity, skipping",
                    entity_id,
                )
                continue
            _LOGGER.debug("force_render dispatching to %s", entity_id)
            hass.async_create_task(entity.async_force_render())
    ```

    **Registration call** (add at end of `async_setup` before `return True`):
    ```python
    hass.services.async_register(
        DOMAIN,
        SERVICE_FORCE_RENDER,
        _async_handle_force_render,
        schema=vol.Schema({vol.Optional(ATTR_ENTITY_ID): cv.entity_ids}),
    )
    ```

    **New imports to add:**
    ```python
    from homeassistant.components.image import ImageEntity as _ImageEntity  # only for isinstance check
    from homeassistant.const import ATTR_ENTITY_ID, Platform
    import voluptuous as vol
    from .image import KrokiImageEntity
    from .const import SERVICE_FORCE_RENDER
    ```

    Note: `ServiceCall` type hint requires `from homeassistant.core import ..., ServiceCall`.
    The existing import is `from homeassistant.core import HomeAssistant` — extend it.

    **IMPORTANT**: Defining `_async_handle_force_render` as a closure inside `async_setup`
    is the idiomatic HA pattern (the reload service uses the same approach internally).
    The handler captures `hass` from the enclosing scope.

    Keep all existing `async_setup` logic (reload service, panel, ws_api) intact.
