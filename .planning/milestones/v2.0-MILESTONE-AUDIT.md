## Critical Gap: PANEL-04 — Broken Method Call

**File:** `custom_components/kroki/ws_api.py:40`

```python
# CURRENT (broken):
rendered = await client.render(msg["diagram_type"], msg["output_format"], msg["source"])

# CORRECT:
rendered = await client.async_render_diagram(
    diagram_type=msg["diagram_type"],
    diagram_source=msg["source"],
    output_format=msg["output_format"],
)
```

**Two problems:**
1. `KrokiClient` has no method `.render()` — correct method is `async_render_diagram()`
2. Argument order is wrong: ws_api passes `(diagram_type, output_format, source)` but signature is `(diagram_type, diagram_source, output_format)`

**Impact:** Any call to `kroki/render` WebSocket command raises `AttributeError` at runtime. The panel live preview (PANEL-04) is non-functional. The Phase 2 VERIFICATION.md did not catch this because `test_panel.py` mocks the client, masking the API contract mismatch.
